GET /api/v1/events/:event_id/comments

Return all comments for event

Supported Formats

json

Errors

Code Description
401 Unauthorized
403 Forbidden

Params

Param name Description
id
required

Event id

Validations:

  • Must be Integer

Headers

Header name Description
access-token
required 		This serves as the user's password for each request. A hashed version of this value is stored in the database for later comparison. This value should be changed on each request.
token-type
required 		Value is always -> 'Bearer'
client
required 		This enables the use of multiple simultaneous sessions on different clients. (For example, a user may want to be authenticated on both their phone and their laptop at the same time.)
expiry
required 		The date at which the current session will expire. This can be used by clients to invalidate expired tokens without the need for an API request.
uid
required 		A unique value that is used to identify the user. This is necessary because searching the DB for users by their access token will make the API susceptible to timing attacks.
